Retail Sales & Inventory Management System – MySQL Project

1️⃣ Project Overview:

Developed a relational retail database using MySQL.

Simulates a real-world retail business with tables for customers, products, orders, order items, inventory movements, and returns.

Focused on database design, normalization, and realistic data generation.

2️⃣ Database Structure:

customers: Stores customer details (name, gender, city, signup date).

products: Stores product information (name, category, price, stock).

categories: Product categories.

orders: Order-level information (customer, date, store).

order_items: Product-level details for each order.

inventory_movements: Tracks stock in/out.

returns: Records returned items and refunds.

3️⃣ Key Features / Highlights:

Normalized database with primary and foreign key relationships.

Realistic data generation for 5,000+ transactions.

Suitable for practicing SQL queries and understanding retail data workflows.

4️⃣ Technologies Used:

MySQL / MySQL Workbench

Relational Database Design

5️⃣ How to Use:

Clone the repository.

Run the provided .sql file in MySQL Workbench to create tables and populate data.
